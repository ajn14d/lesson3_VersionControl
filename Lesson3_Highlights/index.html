Version Control Source Control

Version control is a component of software configuration management. Version control is used to manage changes to documents, programs, web sites ect. Version control is also known as source control is useful for teams that are working on the same project as they can add revisions and work with different versions. Version Control Systems (VCS) are usually run as stand-alone applications. Revision control allows for the ability to revert a document to a previous revision, this is critical for allowing editors to track each other's changes, correct errors, and defend against spamming in wikis.

Git VCS

Git is an implementation of a version control. Git is locally enabled and distributed in a way so that connectivity doesn’t block work. Git has easy to learn commands, and allows many people to contribute to a project and is designed for team collaboration. Git can merge individual’s separate versions into one core version. Essentially Git is a collaborative workstation that allows individuals to contribute and collaborate on a collective project as the same time from different computers. Git can provide all the versions creating allowing individuals to view all of the versions of the material.

Repository File Storage

A repository is an on-disk data structure, this data structure stores metadata for a set of files. It’s possible for the repository to duplicated onto individual user’s system, or it could be housed in a single server. Some of the metadata in the repository could include a historical record of version changes in the repository, set of commit objects, etc. A repository is designed to store files, and remember changes made to the files. Version Control Systems vary in design so the function of the repository can differ between them.

Git Commands Using Git

(GIT INIT) initializes the repository. (GIT REMOTE AND ORIGIN __) tell git to add a remote place called “Origin” whose url is ___. (GIT ADD .) add all the files that were changed since the last back up to the staging area. (GIT COMMIT –M “…”) commit the changes to the repository (GIT PUSH) push the changes in the local repository to the remote repository. (GIT CLONE …) clone a remote repository in … to own local folder. (GIT LOG) show all of the backups created in the repository.

Git Commands Cont.

(GIT STATUS) show all of the files that were changed since the last backup, and which ones are already added to the staging area. (GIT BLAME… ) show who wrote which line of code or who is to blame for that particular line of code. (GIT DIFF) show the differences between the working directory and the backups created in the repository. Git commands were designed to be user friendly and easy to pick-up.
